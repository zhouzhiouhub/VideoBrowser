版本：v1.0  
更新时间：2026-05-25  
项目类型：Android 本地 DIY 浏览器  
核心方向：广告屏蔽 + DOM 去广告 + 视频广告处理 + 视频增强

## 1. 项目定位

VideoBrowser 是一个基于 Android WebView 的本地 DIY 视频增强浏览器。项目重点不是做完整通用浏览器，而是在基础网页浏览能力之上，增强广告处理和视频播放体验。

核心目标：

```text
具备广告屏蔽、DOM 去广告、视频广告处理能力的视频增强浏览器
```

需要实现：

- 普通网页浏览
- 普通网页广告请求拦截
- DOM 广告删除
- CSS 广告隐藏
- JS 动态广告屏蔽
- 自动跳过视频广告
- HTML5 视频倍速
- 视频全屏播放
- 尽量支持后台播放

当前不考虑：

- 应用商店审核
- 商业化
- 云同步
- 用户系统
- 多平台支持
- 自研 Chromium 内核
- DRM 绕过
- 服务端广告拼接破解

## 3. 当前项目状态

当前项目已具备基础 Android 工程结构。

已有内容：

- `MainActivity.kt`
- `WebView` 初始化
- `javaScriptEnabled = true`
- `domStorageEnabled = true`
- `mediaPlaybackRequiresUserGesture = false`
- 默认加载 `https://www.baidu.com`
- Manifest 已声明 `INTERNET` 权限

现阶段问题：

- WebView 配置集中在 `MainActivity`
- 未拆分浏览器核心模块
- 未实现地址栏和搜索
- 未实现广告请求拦截
- 未实现 JS 注入模块
- 未实现全屏视频处理
- 未实现设置系统

后续开发原则：

- `MainActivity` 只负责 UI 绑定和生命周期分发
- WebView 初始化迁移到 `BrowserManager`
- 请求拦截迁移到 `BrowserClient`
- 全屏和进度迁移到 `ChromeClient`
- 广告规则、DOM 处理、JS 注入独立成模块

播放器增强强制原则：

- 能复用播放器、系统或组件库原生能力时，禁止重复实现 UI。
- 优先调用现有 API、复用原生控件、监听已有状态，只补充缺失能力。
- 禁止再创建相同按钮、再绘制播放器 UI Overlay、再维护播放状态、再实现进度系统或 loading 状态。
- 手势层只负责控制播放器行为、调用播放器 API 和提供 HUD 反馈，不负责重绘播放器核心 UI 或替代原生播放器控件。
- 允许自定义快进快退、音量、亮度、倍速、锁屏等 HUD；禁止重复实现播放/暂停按钮、原生进度条、Loading 动画和原生控制栏。
- 核心目标是增强播放器交互，而不是重做播放器。

```text
VideoBrowser
 ├── UI Layer
 ├── Browser Core

 ├── JS Injection Engine
```

```text

   ↓

   ↓
BrowserManager 加载页面
   ↓
BrowserClient 拦截资源请求
   ↓

   ↓

   ↓
页面加载完成
   ↓
JsInjector 注入通用脚本与站点脚本
   ↓
DOM Filter 删除或隐藏广告元素
   ↓
Video Ad Engine 尝试跳广告与视频增强
```

广告处理链路：

```text
Ad Engine
 ├── Request Blocker   请求级拦截
 ├── CSS Filter        样式级隐藏
 ├── DOM Remover       节点级删除
 ├── JS Hook Engine    动态广告拦截
 ├── Video Ad Engine   视频广告处理
 └── Rule Engine       规则解析与匹配
```

## 8. 广告处理系统

### 8.1 广告处理目标

广告处理不是单点能力，而是多层组合：

- 请求发出前屏蔽广告资源
- 页面完成后隐藏广告元素
- 页面完成后删除广告节点
- Hook 动态请求和弹窗
- 针对视频站点做独立适配

执行顺序：

```text
URL 请求拦截
   ↓
页面加载
   ↓
CSS 隐藏
   ↓
DOM 删除
   ↓
JS Hook
   ↓
视频广告处理
```

### 8.2 Request Blocker

功能：

- 拦截广告 JS
- 拦截广告图片
- 拦截 Tracking 请求
- 拦截视频广告请求
- 拦截弹窗脚本

实现位置：


### 8.3 CSS Filter

功能：

- 隐藏广告横幅
- 隐藏浮层广告
- 隐藏信息流广告
- 隐藏暂停广告容器

### 8.4 DOM Remover

功能：

- 删除广告节点
- 删除弹窗
- 删除浮层
- 删除信息流广告
- 监听动态插入的广告节点

示例：

```js
(function () {
  const selectors = [
    '.ad',
    '.ad-banner',
    '.adsbygoogle',
    '.popup-ad',
    '[id*="ad-"]',
    '[class*="advertisement"]'
  ];

  function removeAds() {
    selectors.forEach(function (selector) {
      document.querySelectorAll(selector).forEach(function (element) {
        element.remove();
      });
    });
  }

  removeAds();

  const observer = new MutationObserver(removeAds);
  observer.observe(document.documentElement, {
    childList: true,
    subtree: true
  });
})();
```

### 8.5 JS Hook Engine

功能：

- Hook `fetch`
- Hook `XMLHttpRequest`
- Hook `window.open`
- Hook 部分播放器对象

目标：

- 阻止动态广告请求
- 阻止弹窗广告
- 阻止部分视频广告接口

示例：

```js
(function () {
  if (window.__videobrowser_hooked__) return;
  window.__videobrowser_hooked__ = true;

  const blockedKeywords = ['ad', 'ads', 'advert', 'pagead'];

  function shouldBlock(input) {
    const url = String(input || '').toLowerCase();
    return blockedKeywords.some(function (keyword) {
      return url.includes(keyword);
    });
  }

  const originalFetch = window.fetch;
  window.fetch = function () {
    if (arguments.length > 0 && shouldBlock(arguments[0])) {
      return Promise.reject(new Error('Blocked by VideoBrowser'));
    }
    return originalFetch.apply(this, arguments);
  };

  const originalOpen = window.open;
  window.open = function (url) {
    if (shouldBlock(url)) return null;
    return originalOpen.apply(this, arguments);
  };
})();
```

注意：JS Hook 必须避免破坏正常业务请求。MVP 阶段只使用保守关键字，复杂站点应走 Site Adapter。

### 8.6 Video Ad Engine

目标：

- 处理前贴片广告
- 处理暂停广告
- 处理中插广告
- 设置视频倍速
- 自动点击跳过按钮

处理方式：

1. 自动点击跳过按钮

```js
document.querySelector('.skip, .skip-button, .ytp-ad-skip-button')?.click();
```

2. 视频时长跳跃

```js
if (video.duration && video.duration < 120) {
  video.currentTime = video.duration;
}
```

3. 播放状态增强

```js
video.playbackRate = 2.0;
video.play().catch(function () {});
```

4. 视频广告请求过滤

```text
ad.m3u8
vast
vmap
preroll
midroll
```

限制：

- 同源视频广告可能无法通过 URL 拦截
- 服务端广告拼接无法稳定处理
- DRM 视频无法处理
- 播放器私有状态字段不稳定，不应作为通用方案

## 9. Rule System 设计

### 9.1 规则类型

| 类型 | 示例 | 作用 |
| --- | --- | --- |
| URL 规则 | `||ads.com^` | 请求级拦截 |
| 关键字规则 | `/pagead/` | 请求级拦截 |
| CSS 规则 | `##.ad-banner` | 元素隐藏 |
| DOM 规则 | `remove:.popup-ad` | 元素删除 |
| JS 规则 | `script:common.js` | 脚本注入 |
| Exception 规则 | `@@||example.com^` | 白名单 |
| Site 规则 | `domain:bilibili.com` | 站点适配 |

### 9.2 MVP 支持范围

MVP 只支持：

- URL 包含匹配
- 域名包含匹配
- 简单 CSS 选择器
- 简单 DOM 删除选择器
- 简单白名单

暂不支持：

- 完整 EasyList 语法
- 复杂正则
- `$script`、`$image` 等资源类型修饰符
- `$third-party` 语义
- `:has()` 等复杂选择器

### 9.3 规则优先级

规则优先级从高到低：

1. 用户白名单
2. 站点临时关闭规则
3. Exception 规则
4. 站点专属规则
5. 全局 URL 屏蔽规则
6. 全局 CSS 隐藏规则
7. 全局 DOM 删除规则
8. 全局 JS Hook 规则

执行原则：

- 白名单优先于屏蔽
- 站点规则优先于全局规则
- 请求级拦截优先于页面级处理
- CSS 隐藏优先于 DOM 删除
- JS Hook 必须避免重复注入

## 11. JS Injection 设计

### 11.1 注入时机

基础注入时机：

```text

```

可选增强时机：

- `onPageStarted()` 注入早期 Hook，后期实现
- `onPageFinished()` 注入 CSS、DOM、视频增强脚本

### 11.2 防重复注入

所有脚本必须带唯一标记：

```js
if (window.__videobrowser_common_injected__) return;

```

### 11.3 脚本加载

脚本从 assets 读取：

```kotlin

```

注入：


## 17. 安全设计

### 17.1 WebView 安全配置

禁止默认开启：

```kotlin
settings.allowUniversalAccessFromFileURLs = false
settings.allowFileAccessFromFileURLs = false
```

谨慎配置：

```kotlin
settings.allowFileAccess = false
settings.mixedContentMode = WebSettings.MIXED_CONTENT_COMPATIBILITY_MODE
```

如业务不需要本地文件访问，应保持关闭。

### 17.2 JS Bridge

禁止给不可信网页暴露敏感接口：

```kotlin
addJavascriptInterface()
```

如必须使用：

- 只暴露最小能力
- 不提供文件读写
- 不提供账号信息
- 不提供系统权限能力
- 仅对可信域名启用

### 17.3 Safe Browsing

Android 8.0+ 可开启 Safe Browsing：

```kotlin
WebView.setWebContentsDebuggingEnabled(BuildConfig.DEBUG)
```

调试开关只允许在 Debug 构建中开启。

### 17.4 脚本安全

JS 注入来源：

- 只从本地 assets 加载
- 远程脚本必须校验来源和完整性，MVP 不启用
- 站点脚本应按域名匹配，不应全站点盲注

## 18. 性能优化

### 18.1 WebView 优化

```kotlin
settings.cacheMode = WebSettings.LOAD_DEFAULT
settings.loadsImagesAutomatically = true
settings.blockNetworkImage = false
```

### 18.2 JS 注入优化

必须避免：

- 重复注入同一脚本
- 高频全 DOM 扫描
- 无限制 `setInterval`
- MutationObserver 回调过重

建议：

- 每个脚本使用注入标记
- DOM 删除函数节流
- 选择器数量控制在合理范围
- 站点脚本按域名加载

### 18.3 规则匹配优化

MVP：

- 域名规则使用字符串集合
- 关键字规则使用列表匹配
- 避免每个请求都跑复杂正则

后期：

- 按资源类型分组
- 缓存匹配结果

## 21. 技术难点与限制

### 21.1 视频广告难点

视频广告是最大难点，原因：

- 广告和正片可能同源
- 广告可能由播放器内部控制
- 广告可能服务端拼接进视频流
- 广告接口可能加密或动态签名
- 部分视频受 DRM 保护

可行策略：

- 自动点击跳过按钮
- 拦截明显广告接口
- 删除暂停广告 DOM
- 设置视频倍速
- 站点脚本适配

不可承诺：

- 绕过 DRM
- 稳定跳过所有视频广告
- 处理所有服务端拼接广告

### 21.2 规则维护难点

广告规则频繁变化，需要：

- 规则可配置
- 规则可更新
- 站点可单独适配
- 用户可加入白名单

### 21.3 WebView 能力限制

Android WebView 不是完整 Chrome，可能存在：

- 编码支持差异
- DRM 支持差异
- 自动播放限制
- 后台播放限制
- 某些站点检测 WebView 并限制功能

## 22. 推荐实现顺序

1. 完善主页面 UI：地址栏、WebView、底部栏。
2. 创建 `BrowserManager`，迁移 WebView 初始化。
3. 创建 `UrlUtils`，实现 URL 与搜索词识别。
4. 创建 `BrowserClient`，接管请求拦截和页面完成回调。
5. 创建 `ChromeClient`，处理标题、进度和视频全屏。
6. 创建 `AdBlockManager`，实现基础 URL 黑名单。
7. 创建 `RuleEngine`，封装规则匹配结果。
8. 创建 `JsInjector` 和 `ScriptLoader`，实现 assets 脚本注入。
9. 添加 `common.js`，实现 CSS、DOM、视频增强基础逻辑。
10. 创建 `SiteAdapterRegistry`，按域名加载站点脚本。
11. 创建 `SettingsManager`，保存开关和默认倍速。
12. 添加基础测试页面或测试用例。
13. 测试主流视频站点兼容性。
14. 根据误杀和失效情况调整规则。
