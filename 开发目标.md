# VideoBrowser 开发目标

本文档用于约束后续开发目标和任务顺序。VideoBrowser 不以实现完整 uBlock Origin 引擎为目标，只参考其工程分层、规则编译、索引匹配、决策优先级和安全降级思想，优化本项目的视频浏览和广告处理体验。

---

## 1. 后续执行规则

后续新增功能、重构或修复时，优先按以下顺序执行：

1. 先检查本文档和 `开发流程与进度.md` 中是否存在相关未完成 Markdown 任务。
2. 如果相关 Markdown 任务未完成，先补全文档目标、边界、验收标准和任务拆分。
3. 文档任务完成后，再进入代码实现、测试和验证。
4. 代码实现完成后，必须回写对应 Markdown 任务状态、验证命令和结果。
5. 如果实现过程中发现原目标不合理，先更新 Markdown 设计，再继续改代码。

这条规则的目的不是增加流程负担，而是避免边界不清时直接把复杂能力塞进现有模块。

---

## 2. 总体方向

核心目标：

- 提升视频站点浏览体验。
- 提升广告请求拦截的可解释性和可恢复性。
- 提升规则匹配性能，避免每个请求线性扫描所有规则。
- 支持安全、有限、可测试的页面清理和请求替换能力。
- 未知、高风险或不确定规则默认跳过并记录原因。

非目标：

- 不实现完整 uBlock Origin 引擎。
- 不追求完整 EasyList / AdGuard / uBlock 语法兼容。
- 不执行订阅规则中的任意 JavaScript。
- 不支持任意远程 redirect 资源。
- 不让单个 `Rule` 或 `RuleEngine` 承担所有过滤语义。

---

## 3. 架构参考原则

参考 uBlock 的工程思想，但保持项目自己的能力边界：

- 请求层：负责网络请求判断、阻断、放行、内置 noop 响应。
- 规则层：负责解析安全子集、编译内部模型、建立索引、返回候选规则。
- 决策层：负责用户白名单、站点关闭、放行规则、强制规则和普通规则的优先级。
- 页面层：负责 CSS 隐藏、DOM 删除、跳过按钮、视频增强和安全 Hook 映射。
- 诊断层：负责日志、规则来源、跳过原因、命中原因和误杀恢复入口。

## 4. 阶段目标

### G0 文档门禁

目标：建立后续“先完成 Markdown 任务，再进入实现”的开发约束。

任务：

- `[x]` G0-01 新增本文档。
- `[x]` G0-02 后续每个新阶段开始前，先补齐该阶段任务、边界和验收标准。
- `[x]` G0-03 每次代码实现后，回写对应 Markdown 进度和验证结果。

验收标准：

- 新需求开始前能在 Markdown 中找到明确目标。
- 实现完成后能从 Markdown 追踪任务状态和验证结果。

### G1 请求上下文轻量增强

目标：补齐请求匹配所需上下文，但不模拟完整浏览器扩展 API。

任务：

- `[x]` G1-01 设计 `RequestContext`，包含请求 URL、页面 URL、请求 host、页面 host、主框架标记、方法和请求头。
- `[x]` G1-02 设计 `ResourceType`，支持 `DOCUMENT`、`SCRIPT`、`IMAGE`、`STYLESHEET`、`MEDIA`、`FONT`、`XHR`、`FETCH`、`OTHER`、`UNKNOWN`。
- `[x]` G1-03 新增 `ResourceTypeResolver`，通过 `isForMainFrame`、请求头和 URL 后缀做轻量推断。
- `[x]` G1-04 资源类型不确定时使用 `UNKNOWN`，带资源类型限制的规则默认不命中 `UNKNOWN`。
- `[x]` G1-05 添加单元测试覆盖主文档、脚本、图片、媒体和未知类型。

验收标准：

- 不依赖完整 Chrome 扩展能力。
- 资源类型推断失败时不误杀。
- 现有广告请求拦截行为不回退。

### G2 规则编译与索引

目标：将规则文件解析结果编译为项目内部能力模型，并减少请求时线性扫描。

任务：

- `[x]` G2-01 新增轻量 `RuleCompiler`，把已支持规则转换为内部能力模型。
- `[x]` G2-02 区分请求能力、元素隐藏能力、DOM 删除能力、安全 Hook 能力和 noop 响应能力。
- `[x]` G2-03 为域名规则建立 host / suffix 索引。
- `[x]` G2-04 为 URL contains 规则建立关键词索引。
- `[x]` G2-05 为 CSS / DOM 规则建立页面 host 索引。
- `[x]` G2-06 保留全局规则 fallback。
- `[x]` G2-07 添加大规则文件性能测试。

验收标准：

- 请求匹配不再默认扫描全部规则。
- 索引结果和原有匹配结果一致。
- 未支持规则被跳过并记录原因。

### G3 决策优先级与日志优化

目标：让拦截、放行和误杀恢复更可解释。

任务：

- `[x]` G3-01 新增或拆分 `RuleDecisionResolver`。
- `[x]` G3-02 明确优先级：用户白名单 > 当前站点关闭 > 显式放行 > 强制拦截 > 普通拦截 > 无命中放行。
- `[x]` G3-03 日志记录最终决策、候选规则、覆盖原因和规则来源。
- `[x]` G3-04 日志中保留“加入白名单”和“恢复站点拦截”的操作入口。
- `[x]` G3-05 添加误杀回归测试。

验收标准：

- 白名单和站点关闭永远优先于规则命中。
- 同一请求能解释为什么被拦截或放行。
- 日志能辅助定位规则误杀。

### G4 内置 noop / redirect 响应

目标：只支持项目内置、安全、可测试的替代响应。

任务：

- `[x]` G4-01 新增 `SyntheticResponseFactory`。
- `[x]` G4-02 新增内置资源注册表，只允许 `noopjs`、`noopcss`、`nooptext` 等固定资源。
- `[x]` G4-03 主文档默认禁止 noop / redirect。
- `[x]` G4-04 未知 redirect 资源跳过并记录。
- `[x]` G4-05 添加 WebView 仪器测试，验证广告脚本被 noop 后页面不崩溃。

验收标准：

- 不支持任意远程 redirect。
- 不加载订阅提供的外部替代资源。
- noop 响应 MIME、编码和 body 可测试。

### G5 安全 scriptlet 映射

目标：参考 uBlock scriptlet 的能力形态，但只映射到本地已实现 Hook。

任务：

- `[x]` G5-01 新增 `ScriptletRule` 模型。
- `[x]` G5-02 新增 `ScriptletRegistry`，维护允许的 scriptlet 名称和参数校验。
- `[x]` G5-03 未知 scriptlet 默认跳过并记录。
- `[x]` G5-04 不执行规则原文 JavaScript。
- `[x]` G5-05 支持映射到现有 `common.js` 配置，例如 `window.open` 拦截、fetch 关键字拦截、跳过按钮、视频 controls。
- `[x]` G5-06 添加单元测试和 WebView 仪器测试。

验收标准：

- scriptlet 只能触发本地白名单能力。
- 参数不合法时跳过。
- 不破坏当前站点专属脚本注入。

### G6 回归与发布稳定性

目标：每次扩展广告处理能力时，保持视频播放、导航和站点开关稳定。

任务：

- `[x]` G6-01 建立广告拦截回归测试集合。
- `[x]` G6-02 建立视频播放和全屏回归测试集合。
- `[x]` G6-03 建立站点关闭拦截、关闭 JS 注入和用户白名单回归测试。
- `[x]` G6-04 建立规则加载失败和缓存回退测试。
- `[x]` G6-05 每个阶段记录构建、单测、仪器测试结果。

验收标准：

- 新增过滤能力不影响基础浏览。
- 视频增强能力不因规则层调整回退。
- 失败规则不会导致 App 崩溃。
