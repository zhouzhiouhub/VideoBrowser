# Function Entry And Quick Shortcuts Design

## Goal

保留当前百度风格浏览器外壳，但去掉账号化内容。底部原“我的”入口继续保留，语义改为“功能”，用于展示浏览器工具和增强入口，不再展示登录、手机号、会员、红包或任何个人账号相关内容。

## Scope

- 底部头像/“我的”入口改为功能入口，文案和无障碍描述改成“功能”。
- 半屏菜单移除登录样式和个人中心头部，只展示百度风格的浏览器可用内容。
- 独立功能页保留，但内容改为浏览器功能集合：历史、收藏、文件、拦截日志、浏览器增强和必要设置入口。
- 搜索栏下方恢复不同搜索/浏览器入口，并保持当前百度风格圆形快捷入口视觉。
- 搜索栏下方增加一个加号入口，用于添加自定义快捷入口。
- 自定义快捷入口最多显示 10 个；超过 10 个时，新添加入口保留，最早添加的自定义入口被挤出。

## Out Of Scope

- 不实现真实登录、账号同步、会员、红包或手机号识别。
- 不删除收藏、历史、站点设置、清理数据、白名单等底层能力。
- 不实现多标签页或完整浏览器首页重建。
- 不把自定义入口同步到云端。

## UI Behavior

主浏览器界面保留顶部百度风格搜索栏和底部导航。底部最后一个入口仍在原位置，但图标和文案表达“功能”，点击后打开独立功能页。

半屏菜单只显示与当前网页和浏览器相关的可用操作，例如分享、收藏、历史、文件、刷新、当前站点和更多功能入口。它不再显示“立即登录”头部，也不再跳转到账号化页面。

功能页作为原个人中心页的替代，使用同一套视觉样式，但首屏不再有手机号、登录按钮、会员卡、红包卡。页面内容按浏览器功能组织，保留已经可用的工具和设置，不引入新的账号概念。

搜索栏下方显示默认搜索/浏览器入口和用户快捷入口。默认入口继续来自现有 `SearchProviders.defaults`。加号入口打开添加对话框，用户输入名称和网址后保存。添加成功后该入口出现在横向列表末尾；如果自定义入口超过 10 个，删除最早保存的自定义入口。

## Data Flow

默认搜索/浏览器入口继续由 `SearchProviderController` 和 `SearchProviders` 管理。

自定义快捷入口存入 `SettingsManager` 背后的 `PreferenceStore`。数据结构保存名称和 URL，读取时过滤空名称、非法 URL 和重复无效项。添加入口时按插入顺序维护列表，超过限制时从列表头部裁剪。

点击默认入口仍切换搜索提供方并打开对应首页。点击自定义快捷入口直接加载保存的 URL，不改变默认搜索提供方。

## Error Handling

添加快捷入口时，如果名称为空、URL 为空或 URL 不是 `http` / `https`，显示简短错误提示并不保存。

如果保存数据损坏，读取时丢弃损坏项，保留有效入口。

## Testing

单元测试覆盖自定义快捷入口的保存、读取、最多 10 个裁剪、非法 URL 过滤，以及默认搜索提供方不受自定义入口影响。

Android 构建验证资源、布局和 Kotlin wiring。
