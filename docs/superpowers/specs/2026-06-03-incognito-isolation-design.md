# 无痕隔离与首页显示设计

日期：2026-06-03

## 背景

当前浏览器已经有无痕开关、历史记录、收藏夹、搜索引擎横栏、首页按钮和无痕徽标。现有无痕开关是持久化设置，会直接作用在当前 WebView 上，并在切换时清理浏览数据和历史记录。这会影响标准模式的状态，也会让应用下次启动继续处于无痕状态，不符合“标准模式和无痕模式互不影响”以及“无痕模式应用退出时自动退出，不进行持久化”的要求。

## 目标

- 首页状态不显示底部“首页”入口，不显示顶部网页收藏按钮，不显示无痕徽标，也不显示“标准/无痕”模式文案。
- 无痕开启时界面使用深色主题。
- 无痕模式只在当前应用进程或当前 Activity 生命周期内有效，不写入偏好设置；应用退出后自动结束，下次启动固定进入标准模式。
- 无痕会话中的浏览操作不写入历史记录，也不写搜索栏下方展示内容、搜索引擎选择、首页地址、站点设置、白名单和元素屏蔽规则。
- 无痕下允许用户主动收藏当前网页，收藏写入标准收藏夹；这是无痕模式唯一允许写入的持久化用户数据。
- 无痕模式与标准模式隔离：切入无痕时不破坏标准模式当前页面、回退栈、设置、历史、收藏；切回标准时恢复标准模式原状态。
- 无痕退出或应用销毁时清理无痕会话产生的 Cookie、缓存、WebStorage、WebView 表单数据、SSL 状态和 WebView 历史。

## 非目标

- 不新增账号同步、云收藏或跨设备隐私能力。
- 不修改视频增强、广告拦截、下载、分享、原生播放器等功能的核心行为。
- 不清空标准模式已有收藏、历史记录或浏览器设置。

## 推荐方案

使用独立浏览会话实现标准/无痕隔离。标准模式保留当前 WebView 与控制器状态；无痕模式创建独立的临时 WebView，会话结束后销毁。当前模式保存在运行时内存中，不写入 `SharedPreferences`。界面层根据当前模式决定可见控件、主题和可写行为。标准模式和无痕模式共享只读能力，例如内置规则、脚本资源、搜索引擎列表；所有会改变用户配置或浏览记录的写操作都必须经过当前模式判断。

## 组件设计

### BrowsingSessionMode

新增模式枚举：

- `STANDARD`：标准模式，使用持久化设置和标准 SavedPageRepository，也是每次应用启动的默认模式。
- `PRIVATE`：无痕模式，使用运行时临时会话状态，不写入持久化浏览记录、模式状态和用户配置；用户主动收藏当前页时例外，写入共享收藏夹。

### PrivateBrowsingSession

新增无痕会话协调对象，负责：

- 保存当前模式。
- 保证当前模式只存在于内存中，不调用 `SettingsManager.setPrivateBrowsingEnabled` 之类的持久化写入。
- 切入无痕时创建临时 WebView，配置独立 BrowserManager。
- 切回标准时销毁无痕 WebView 并恢复标准 WebView。
- 对外暴露 `isPrivate`、`activeWebView`、`activeBrowserManager`。
- 在用户关闭无痕、Activity 销毁或应用退出时调用清理逻辑，并把运行时模式恢复为 `STANDARD`。

标准 WebView 不销毁，只从容器中临时移除或隐藏，因此标准模式的当前页面和回退栈能保持。

### 记录写入策略

所有可能写入用户状态的入口必须检查当前模式：

- 无痕开关：只切换运行时会话，不写 `private_browsing` 偏好；如保留旧偏好键，启动时也必须忽略并迁移为标准模式。
- 历史记录：无痕下不调用 `SavedPageRepository.addHistory`。
- 收藏：无痕下允许用户主动收藏或取消收藏当前网页，写入同一个收藏夹；该动作不生成历史记录。
- 收藏夹打开记录：无痕模式打开收藏夹条目时仍不写历史；只有标准模式打开收藏夹条目并完成页面加载时，才按普通标准浏览写入历史记录。
- 搜索引擎选择和首页 URL：无痕首页不显示搜索引擎横栏；无痕下不写 `search_provider` 和 `home_url`。
- 站点规则、白名单、元素屏蔽：无痕下不展示会产生持久配置的入口。
- 清理浏览数据、恢复默认设置：仍然只影响标准模式已有语义，不因为无痕切换额外清空标准数据。

### 首页显示规则

当 `isHomePageVisible == true`：

- 底部首页按钮隐藏。
- 顶部收藏按钮隐藏。
- 无痕徽标隐藏。
- 不显示“标准模式”“无痕模式”“回到首页”“网页收藏”一类模式或首页说明文案。
- 标准模式首页可以继续显示搜索引擎横栏。
- 无痕模式首页不显示搜索引擎横栏，避免把无痕操作反映到搜索栏下方的标准内容。

当打开普通网页时：

- 底部首页按钮可以显示，用于回首页。
- 标准模式显示收藏按钮。
- 无痕模式显示收藏按钮，用于用户主动写入收藏夹；无痕模式不显示无痕徽标。

### 主题规则

- 标准模式使用应用当前默认主题。
- 无痕模式进入时应用夜间/深色颜色集合。
- 切回标准模式时恢复标准模式主题，不把无痕深色选择写入持久设置。

如果直接使用全局 night mode 会导致 Activity 重建并影响标准模式状态，则改为在根布局和控件上应用当前模式颜色，避免重建。

## 数据流

1. 应用启动始终进入标准模式，初始化标准 WebView、BrowserManager 和功能中心；即使旧版本曾保存无痕偏好，也不恢复无痕模式。
2. 用户在浏览器设置中开启无痕。
3. Session 协调对象隐藏或移除标准 WebView，创建并显示无痕 WebView。
4. 无痕 WebView 打开默认搜索首页或空首页，UI 切到深色，并隐藏首页按钮、收藏按钮、无痕徽标和搜索引擎横栏。
5. 无痕浏览期间，页面完成加载不会写历史；用户主动收藏当前网页时写入收藏夹，但功能中心不会暴露站点持久配置、白名单、元素屏蔽等写入入口。
6. 用户关闭无痕时，Session 协调对象清理并销毁无痕 WebView，恢复标准 WebView、标准 UI 和标准当前页面。
7. 如果用户直接退出应用，Activity 销毁时清理并销毁无痕会话；下次启动进入标准模式。

## 错误处理

- 无痕 WebView 创建失败时保持标准模式，不改变标准数据，并提示无痕启动失败。
- 无痕清理 Cookie、缓存或 WebStorage 失败时不阻塞切回标准，但仍销毁无痕 WebView 并刷新 UI。
- 切换模式时如果页面处于全屏或元素屏蔽选择状态，先退出这些临时 UI 状态，再切换会话。

## 测试计划

单元测试：

- 无痕模式下历史写入被跳过。
- 无痕模式下用户主动收藏会写入收藏夹，但不会写入历史记录。
- 无痕模式打开收藏夹条目不会写入历史；标准模式打开收藏夹条目会写入历史。
- 搜索引擎选择在无痕下不写入 `SettingsManager`。
- 无痕切换不清空标准历史和收藏。
- 无痕开关不写入 `private_browsing` 偏好；应用重启后默认标准模式。

界面/集成验证：

- 首页隐藏底部首页按钮、顶部收藏按钮和无痕徽标。
- 标准模式首页显示搜索引擎横栏；无痕模式首页不显示搜索引擎横栏。
- 无痕打开网页后不产生历史记录，切回标准后历史记录保持原样。
- 无痕打开网页后点击收藏，收藏夹出现该网页；历史记录仍不出现该网页。
- 无痕打开收藏夹条目后不产生历史；标准模式打开同一收藏夹条目后产生历史。
- 切入无痕后界面为深色，切回标准后恢复标准界面。
- 标准页面打开后切入无痕，再切回标准，原页面和回退能力保留。
- 无痕状态下退出应用再启动，首页和设置均处于标准模式。

## 验收标准

- 标准模式和无痕模式的页面状态、历史和设置互不影响；收藏夹是用户主动收藏时共享写入的唯一例外。
- 无痕模式不持久化，应用退出后自动结束，下次启动不会继续处于无痕模式。
- 无痕期间不出现可写入历史、设置或站点规则的入口；收藏入口只在用户主动收藏当前页时写入收藏夹。
- 首页不展示用户要求隐藏的首页、收藏、无痕/标准相关内容。
- 自动化测试通过，Android 构建通过。
